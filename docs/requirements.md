## 1. 项目概述

**Academic Figure Generator Desktop** 是一款面向学术研究人员的本地桌面应用，帮助用户基于论文内容自动生成专业学术配图。

### 背景

原 Web 版本（React 前端 + FastAPI 后端）需要服务器部署和维护，存在以下问题：
- 用户需要依赖中心化服务器
- API Key 存储在服务端，存在隐私顾虑
- 部署和运维成本高
- 无法离线使用

### 改造目标

将 Web 版改造为**本地独立桌面应用**，具备以下特性：
- 使用 **Tauri 2.0** 作为桌面框架，复用已有的 React 前端代码
- **完全客户端化**：无需服务器，所有数据本地存储
- **用户自备 API Key**：Claude API、NanoBanana API 等均由用户自行配置
- **完全免费开源**：MIT 协议开源

## 2. 目标用户

### 主要用户群体

- **学术研究人员**：高校教师、科研院所研究员
- **硕博士研究生**：正在撰写学位论文或投稿顶会的学生

### 使用场景

- 为学术论文生成配套插图：
  - 系统架构图（overall framework）
  - 网络结构图（network architecture）
  - 模块细节图（module detail）
  - 对比与消融实验图（comparison & ablation）
  - 数据行为可视化图（data behavior）
- 面向顶会/期刊投稿场景，配图需符合专业学术审美：
  - CVPR、NeurIPS、ICCV、ECCV（计算机视觉）
  - ICML、ICLR（机器学习）
  - Nature、Science（综合科学）
  - IEEE Transactions 系列（工程技术）

### 用户痛点

1. 手动绘制学术配图耗时耗力，工具学习成本高（Visio、Illustrator 等）
2. AI 生成图片质量难以控制，生成的图不符合学术审美
3. 不同论文需要统一配色风格，手动维护麻烦
4. 对数据隐私敏感，不愿将论文内容上传至第三方服务器

## 3. 功能模块

### 3.1 初始设置 (First Run Setup)

**触发时机**：应用首次启动，检测到未配置 API Key 时显示引导页。

**配置项**：

| 配置项 | 必填 | 说明 |
|--------|------|------|
| Claude API Key | 必填 | Anthropic Claude API 密钥 |
| Claude Base URL | 可选 | 默认 `https://api.anthropic.com`，支持代理 |
| Claude 模型选择 | 必填 | 从可用模型列表中选择，默认 claude-opus-4-5 |
| NanoBanana API Key | 必填 | 图片生成服务 API 密钥 |
| NanoBanana Base URL | 可选 | 默认官方地址，支持自定义 endpoint |
| PaddleOCR Server URL | 可选 | 用户自建 OCR 服务地址 |
| PaddleOCR Token | 可选 | OCR 服务认证 Token |

**存储方式**：配置存储在本地加密 Key-Value 存储中（Tauri `stronghold` 或系统 keychain），不以明文存储。

**引导流程**：
1. 欢迎页面 → 说明应用功能
2. Claude API 配置（必填）
3. NanoBanana API 配置（必填）
4. OCR 服务配置（可跳过）
5. 完成 → 进入主界面

### 3.2 项目管理

每个**项目**对应一篇论文或一组相关配图需求。

**创建项目时填写**：

| 字段 | 必填 | 说明 |
|------|------|------|
| 项目名称 | 必填 | 如 "CVPR 2026 主论文" |
| 描述 | 可选 | 简短备注 |
| 研究领域 | 可选 | 如 "计算机视觉 / 目标检测" |
| 配色方案 | 必填 | 从预设或自定义配色中选择 |

**项目列表功能**：
- 按最后更新时间倒序排列
- 显示项目名称、文档数量、图片数量、最后更新时间
- 支持搜索/筛选

**项目操作**：
- 编辑项目信息（名称、描述、配色方案）
- 删除项目（同时删除关联文档和图片，需二次确认）
- 进入项目工作区

**项目内容**：
- 上传的文档列表
- 生成的提示词列表
- 生成的图片列表

### 3.3 文档上传与解析

**支持格式**：

| 格式 | 解析库 | 说明 |
|------|--------|------|
| PDF | pdf.js | 纯客户端，提取文本层 |
| DOCX | mammoth.js | 转换为结构化文本 |
| TXT | 原生处理 | 按段落/章节分割 |

**解析结果**：
- 全文文本（用于提示词生成）
- 章节结构（标题树，便于用户选择特定章节）
- 页数/字数统计

**可选 OCR 增强**：
- 当 PDF 为扫描版（无文本层）时，可调用用户配置的 PaddleOCR 服务器
- OCR 结果合并到解析文本中
- OCR 调用为可选，不影响基础功能

**文件存储**：
- 原始文件存储在应用数据目录（如 `~/Library/Application Support/academic-figure-generator/documents/`）
- 解析结果存储在 SQLite 数据库中
- 支持删除文档（同时清理本地文件）

---

### 3.4 提示词生成

基于论文内容，调用 Claude API 自动生成专业的图片生成提示词。

**生成参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| 文档/章节选择 | 必选 | 选择一个或多个章节作为上下文 |
| 图表类型 | 多选 | 见下方图表类型列表 |
| 配色方案 | 必选 | 当前项目配色 |
| 用户自定义请求 | 可选 | 额外说明，如"重点突出模块间的数据流向" |
| 最大生成数量 | 必填 | 默认 3，范围 1-10 |
| 模板模式 | 开关 | 开启后无需文本输入，使用纯结构模板 |

**支持的图表类型**：

| 类型 ID | 中文名称 | 说明 |
|---------|----------|------|
| `overall_framework` | 总体框架图 | 展示系统整体架构和模块关系 |
| `network_architecture` | 网络架构图 | 深度学习网络结构细节 |
| `module_detail` | 模块细节图 | 单个模块内部工作原理 |
| `comparison_ablation` | 对比消融图 | 不同方法/配置的对比展示 |
| `data_behavior` | 数据行为图 | 数据流动、分布、变换可视化 |

**生成结果**：
- 每个提示词包含：英文 prompt 正文、图表类型标签、生成时间
- 结果可编辑（直接在界面修改文本）
- 结果可删除
- 结果保存到 SQLite 数据库，与项目关联

---

### 3.5 图片生成

基于提示词调用 NanoBanana API 生成学术配图。

**生成参数**：

| 参数 | 选项 | 默认值 |
|------|------|--------|
| 分辨率 | 1K / 2K / 4K | 2K |
| 宽高比 | 1:1, 4:3, 3:4, 16:9, 9:16, 3:2, 2:3 | 4:3 |

**生成模式**：

1. **从提示词生成**：选择已有提示词 → 调整参数 → 生成
2. **直接生成模式**（对应「直接生成」页面）：自由输入 prompt → 生成，不关联到任何项目/提示词

**图片管理**：
- 图片保存到本地文件系统（应用数据目录下的 `images/` 子目录）
- 支持预览（缩略图 + 全尺寸查看）
- 支持下载/导出到用户指定位置
- 支持删除
- 显示生成参数（分辨率、宽高比、所用提示词）

**错误处理**：
- API 调用失败时显示具体错误信息
- 支持重试
- 余额不足时提示用户充值

---

### 3.6 图片编辑（图生图）

基于参考图片进行风格迁移或内容修改。

**操作流程**：
1. 上传参考图片（支持 PNG、JPEG、WebP）
2. 输入编辑指令（中英文均可）
3. 选择生成参数（分辨率、宽高比）
4. 调用 NanoBanana 图生图 API
5. 预览结果，满意后保存

**使用场景**：
- 基于草图生成精细化配图
- 统一多张图的视觉风格
- 局部修改已有配图

---

### 3.7 配色方案管理

学术配图对配色要求严格，需兼顾美观性和色盲友好性。

**9 种内置预设配色方案**：

| 方案名 | 说明 |
|--------|------|
| Okabe-Ito | 色盲友好，国际学术推荐方案（**默认推荐**） |
| Blue Monochrome | 蓝色单色系，适合严肃技术报告 |
| Warm Earth | 暖土色系，自然亲和 |
| Purple-Green | 紫绿对比，视觉冲击力强 |
| Grayscale | 灰度方案，适合黑白印刷 |
| Teal-Coral | 蓝绿珊瑚色，现代感强 |
| ML TopConf Tab10 | 顶会论文常用 Tab10 调色板 |
| ML TopConf Colorblind | 顶会论文色盲友好版 |
| ML TopConf Deep | 顶会论文深色版 |

**自定义配色方案**：

每个配色方案定义 8 种语义角色颜色：

| 角色 | 说明 |
|------|------|
| Primary | 主色，重点元素 |
| Secondary | 次色，辅助元素 |
| Accent | 强调色，高亮/焦点 |
| Background | 背景色 |
| Text | 正文颜色 |
| Border | 边框/分隔线 |
| Success | 正向状态 |
| Warning | 警告状态 |

**功能**：
- 查看预设方案色板预览
- 创建/编辑自定义方案（颜色拾取器）
- 删除自定义方案
- 每个项目独立选择配色方案

---

### 3.8 用量统计

本地记录 API 调用情况，帮助用户了解消耗。

**统计维度**：

| 统计项 | 说明 |
|--------|------|
| Claude API Token 用量 | 按月统计输入/输出 token 数，估算费用 |
| NanoBanana 图片生成次数 | 按月统计，细分分辨率档位（1K/2K/4K） |
| 按项目汇总 | 各项目的资源消耗占比 |

**展示方式**：
- 本月用量概览（数字卡片）
- 近 6 个月趋势图（折线图/柱状图）
- 按日期明细列表

**数据来源**：
- 每次 API 调用成功后，在本地 SQLite 数据库记录调用日志
- 完全本地，不上传任何数据

## 4. 非功能需求

### 4.1 跨平台支持

| 平台 | 支持状态 |
|------|----------|
| macOS (Apple Silicon M1/M2/M3/M4) | 正式支持 |
| macOS (Intel x86_64) | 正式支持 |
| Windows 10 / 11 (x86_64) | 正式支持 |
| iOS | 后续版本（v2.0） |
| Android | 后续版本（v2.0） |

### 4.2 性能要求

| 指标 | 目标 |
|------|------|
| 冷启动时间 | < 3 秒 |
| 安装包体积 | < 30 MB |
| 内存占用（空闲时） | < 150 MB |
| 文档解析（10MB PDF） | < 10 秒 |

### 4.3 离线可用性

- **完全离线**：项目管理、文档查看、提示词编辑、历史图片浏览
- **需要网络**：调用 Claude API（提示词生成）、调用 NanoBanana API（图片生成）、可选 OCR 服务
- 无网络时，已有数据仍可正常访问和编辑

### 4.4 数据安全

- API Key 使用系统级加密存储（macOS Keychain / Windows Credential Manager，通过 Tauri `stronghold` 插件）
- 不上传任何用户数据到第三方（API 调用参数除外，且直接发送给用户指定的 API endpoint）
- 本地数据库不加密（仅存储非敏感的项目数据），API Key 单独加密存储

### 4.5 可用性

- 支持深色模式 / 浅色模式（跟随系统设置）
- 支持中文界面（默认）
- 界面响应时间 < 100ms（本地操作）

## 6. 用户流程

### 主流程（论文配图全流程）

```
首次启动
   └─→ 初始设置引导（/setup）
          └─→ 配置 Claude API Key
          └─→ 配置 NanoBanana API Key
          └─→ [可选] 配置 PaddleOCR
          └─→ 完成 → 跳转项目列表

项目列表 (/projects)
   └─→ 新建项目（填写名称、研究领域、配色方案）
   └─→ 进入项目工作区 (/projects/:id)

项目工作区
   ├─→ [文档标签页]
   │      └─→ 上传 PDF/DOCX/TXT
   │      └─→ 自动解析 → 显示章节结构
   │      └─→ [可选] OCR 增强
   │
   ├─→ [提示词标签页]
   │      └─→ 选择章节 + 图表类型 + 自定义要求
   │      └─→ 调用 Claude API → 生成提示词列表
   │      └─→ 查看 / 编辑 / 删除提示词
   │
   └─→ [图片标签页]
          └─→ 选择提示词 → 选择分辨率/宽高比
          └─→ 调用 NanoBanana API → 生成图片
          └─→ 预览图片 → 下载/导出
          └─→ [可选] 图生图编辑
```

### 快速流程（直接生成图片）

```
直接生成页面 (/generate)
   └─→ 输入/粘贴 prompt 文本
   └─→ 选择分辨率和宽高比
   └─→ [可选] 上传参考图（图生图模式）
   └─→ 点击生成
   └─→ 预览 → 下载
```

### 配色管理流程

```
配色方案页面 (/color-schemes)
   ├─→ 查看 9 种预设方案的色板预览
   └─→ 新建自定义方案
          └─→ 命名方案
          └─→ 配置 8 种语义颜色
          └─→ 保存
          └─→ 在项目中选用
```

## 7. 数据存储需求

### 存储结构

```
~/Library/Application Support/academic-figure-generator/   (macOS)
%APPDATA%\academic-figure-generator\                       (Windows)
├── database.sqlite          # 结构化数据（项目、提示词、用量记录）
├── documents/               # 上传的原始文档
│   └── {project_id}/
│       └── {document_id}.{ext}
└── images/                  # 生成的图片
    └── {project_id}/
        └── {image_id}.png
```

### SQLite 数据库主要表

| 表名 | 内容 |
|------|------|
| `projects` | 项目基本信息 |
| `documents` | 文档元数据和解析结果 |
| `prompts` | 生成的提示词 |
| `images` | 生成的图片元数据（文件路径等） |
| `color_schemes` | 自定义配色方案 |
| `api_usage_logs` | API 调用日志（用于用量统计） |
| `settings` | 用户偏好设置（非敏感） |

### 加密存储

- **API Key 存储**：使用 Tauri `stronghold` 插件或系统 Keychain（macOS）/ Credential Manager（Windows）
- **存储内容**：Claude API Key、NanoBanana API Key、PaddleOCR Token
- **不加密**：项目数据、文档解析结果、图片文件（存储在用户本地，已受 OS 权限保护）
